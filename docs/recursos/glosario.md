# Glosario

## Términos de ISO 27001

- **AP (Action Plan)** - Plan de acción que traduce el RTP en tareas ejecutables con responsables, fechas y recursos
- **CIA** - Confidencialidad, Integridad y Disponibilidad (Confidentiality, Integrity, Availability)
- **ISMS (Information Security Management System)** - Sistema de Gestión de Seguridad de la Información (SGSI en español)
- **ISO** - International Organization for Standardization (Organización Internacional de Normalización)
- **IEC** - International Electrotechnical Commission (Comisión Electrotécnica Internacional)
- **RTP (Risk Treatment Plan)** - Plan de Tratamiento de Riesgos que documenta decisiones estratégicas sobre cada riesgo identificado
- **SOA (Statement of Applicability)** - Declaración de Aplicabilidad que justifica qué controles del Anexo A se implementan y por qué

## Términos de Gestión de Riesgos

- **Riesgo** - Cualquier aspecto que amenace o restrinja la capacidad de una organización de lograr su objetivo
- **Activo** - Cualquier recurso con valor para la organización (datos, sistemas, infraestructura, personas)
- **Amenaza** - Causa potencial de incidente no deseado que puede causar daño a un activo
- **Impacto** - Consecuencia de la materialización de un riesgo (financiero, reputacional, operacional, legal)
- **IDOR (Insecure Direct Object Reference)** - Vulnerabilidad que permite acceder a objetos sin autorización adecuada
- **Probabilidad** - Posibilidad de que una amenaza se materialice
- **Magnitud** - Combinación de la probabilidad de un evento y su impacto (Magnitud = Probabilidad × Impacto)
- **Riesgo Inherente** - Riesgo antes de aplicar controles de mitigación
- **Riesgo Residual** - Riesgo que permanece después de aplicar controles
- **Vulnerabilidad** - Debilidad que puede ser explotada por una amenaza
- **Gestión de riesgos de seguridad de la información** - Proceso continuo de identificar, resolver y prevenir problemas de seguridad
- **Evaluación de riesgos** - Proceso para identificar vulnerabilidades y amenazas a los recursos de datos usados por una organización para lograr sus objetivos de negocio y determinar contramedidas para reducir su nivel de aceptación. 

## Términos de Desarrollo Seguro

- **DAST (Dynamic Application Security Testing)** - Pruebas de seguridad dinámicas en aplicaciones en ejecución
- **OWASP** - Open Web Application Security Project (Proyecto Abierto de Seguridad en Aplicaciones Web)
- **RBAC (Role-Based Access Control)** - Control de Acceso Basado en Roles
- **SAST (Static Application Security Testing)** - Análisis estático de código fuente para detectar vulnerabilidades
- **SCA (Software Composition Analysis)** - Análisis de componentes de terceros y dependencias
- **Secure SDLC** - Ciclo de Vida de Desarrollo de Software Seguro
- **SQL Injection** - Inyección de código SQL malicioso en consultas de base de datos
- **XSS (Cross-Site Scripting)** - Inyección de scripts maliciosos en páginas web

## Términos de Logging y Monitoring

- **Correlation ID** - Identificador único para rastrear requests a través de múltiples servicios
- **KQL (Kusto Query Language)** - Lenguaje de consulta para Azure Application Insights y otros servicios
- **Log Retention** - Período de tiempo que se conservan los logs (típicamente 90-365 días)
- **Serilog** - Biblioteca de logging estructurado para .NET
- **SIEM (Security Information and Event Management)** - Sistema de gestión de información y eventos de seguridad
- **Sink** - Destino de logs en Serilog (consola, archivo, Azure, etc.)
- **Structured Logging** - Logging con propiedades estructuradas (no solo texto plano)

## Opciones de Tratamiento de Riesgos

- **Modificar (Mitigate)** - Reducir el riesgo implementando controles de seguridad
- **Retener (Accept)** - Aceptar el riesgo con justificación formal y aprobación de dirección
- **Evitar (Avoid)** - Eliminar la actividad que genera el riesgo
- **Compartir (Transfer/Share)** - Transferir el riesgo a terceros (seguros, outsourcing, servicios cloud)
